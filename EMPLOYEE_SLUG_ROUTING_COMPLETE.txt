HOW IT WORKS
================================================================================

1. EMPLOYEE LOGIN:
   - Employee logs in with email/phone and password
   - System generates slug: {name}-{employee_code}
   - Example: "alok-CCEMP0780"
   - Redirects to: /employee/alok-CCEMP0780/dashboard

2. SLUG VALIDATION:
   - EmployeeAuthGuard checks if URL slug matches logged-in employee
   - If mismatch: redirects to correct employee slug
   - If no employee: redirects to /employee/login

3. NAVIGATION:
   - All sidebar links include employee slug
   - All buttons and actions include employee slug
   - All redirects include employee slug
   - Profile dropdown includes employee slug

4. SECURITY:
   - Cannot access another employee's routes
   - Cannot access routes without proper slug
   - Automatic validation on every route
   - Session-based authentication

EMPLOYEE SLUG FORMAT
================================================================================

SLUG GENERATION:
  - Full Name: "Alok Kumar"
  - Employee Code: "CCEMP0780"
  - Generated Slug: "alok-kumar-CCEMP0780"

RULES:
  - Lowercase name
  - Spaces replaced with hyphens
  - Special characters removed
  - Employee code appended with hyphen
  - Unique per employee

EXAMPLES:
  - "Rahul Singh" + "CCEMP0001" → "rahul-singh-CCEMP0001"
  - "Priya Sharma" + "CCEMP0002" → "priya-sharma-CCEMP0002"
